SQL

SQL, que significa Structured Query Language (Linguagem de Consulta Estruturada), é uma linguagem de programação utilizada para gerir e manipular bases de dados.
É particularmente útil para tarefas como a consulta de dados, a criação e modificação de estruturas de bases de dados e o controlo do acesso aos dados armazenados nas bases de dados.

phpMyAdmin

O phpMyAdmin é uma ferramenta de administração gratuita e de código aberto para bases de dados MySQL e MariaDB, escrita em PHP, que foi criada por Tobias Ratschiller em 1998. A ferramenta tornou-se uma das ferramentas de administração MySQL mais populares, especialmente para serviços de alojamento Web. Em 2001, um grupo de três programadores registou o projeto phpMyAdmin no SourceForge e assumiu o desenvolvimento. Desde então, o software passou por várias atualizações e melhorias, incluindo a utilização de Ajax para melhorar a usabilidade. Com o phpMyAdmin, é possível executar a maioria das tarefas de administração, incluindo a criação de uma base de dados e a execução de consultas.

Algumas das características do phpMyAdmin incluem a capacidade de criar, editar e eliminar bases de dados, tabelas, vistas, colunas e índices. Também pode apresentar vários conjuntos de resultados através de procedimentos armazenados ou consultas, criar, copiar, eliminar, renomear e alterar bases de dados, tabelas, colunas e índices, e executar e editar qualquer instrução SQL.

Algumas das vantagens do phpMyAdmin incluem a sua interface de fácil utilização, que permite aos utilizadores gerir facilmente as suas bases de dados sem necessidade de quaisquer conhecimentos técnicos. Além disso, o phpMyAdmin fornece uma gama de funcionalidades, tais como cópia de segurança e restauro da base de dados, pesquisa de bases de dados e criação de consultas, o que pode poupar aos programadores uma quantidade significativa de tempo na gestão das suas bases de dados. É também altamente personalizável, com uma gama de temas e plugins disponíveis para melhorar a sua funcionalidade.

No entanto, uma das principais desvantagens da utilização do phpMyAdmin é a segurança. Sendo uma aplicação baseada na Web, o phpMyAdmin é vulnerável a ataques como a injeção de SQL e o cross-site scripting. Isto significa que se um atacante obtiver acesso à sua instalação do phpMyAdmin, pode potencialmente causar estragos na sua base de dados. Além disso, pode ser utilizado por alguém com intenções maliciosas para roubar dados ou comprometer a sua aplicação PHP. Por isso, é importante garantir que a sua instalação do phpMyAdmin está devidamente protegida e que está a monitorizá-la para detetar qualquer atividade suspeita.

Existem várias alternativas ao phpMyAdmin para gerir bases de dados MySQL. Uma opção popular é o Adminer, que é uma ferramenta de gestão de bases de dados leve e fácil de utilizar. Outra opção é o Sequel Pro, que é uma ferramenta apenas para Mac que oferece uma variedade de funcionalidades poderosas para gerir bases de dados MySQL. Adicionalmente, alguns programadores podem optar por utilizar ferramentas de linha de comandos como o MySQL Workbench ou o pgAdmin, que proporcionam uma abordagem mais prática à gestão de bases de dados.

Tipos de Dados

O SQL suporta vários tipos de dados para armazenar informações em tabelas. Abaixo estão alguns dos tipos de dados mais comuns:

  • INT: Armazena números inteiros. Ex: 10, -25, 0.
  • VARCHAR(n): Armazena strings de tamanho variável, com um limite máximo de caracteres definido por 'n'. Ex: VARCHAR(255), VARCHAR(50).
  • CHAR(n): Armazena strings de tamanho fixo, com um limite máximo de caracteres definido por 'n'. Ex: CHAR(10), CHAR(20).
  • TEXT: Armazena strings de texto longo. Adequado para textos extensos como descrições. Não tem limite definido.
  • DATE: Armazena uma data no formato 'YYYY-MM-DD'. Ex: '2024-02-16'.
  • DATETIME: Armazena data e hora no formato 'YYYY-MM-DD HH:MM:SS'. Ex: '2024-02-16 14:30:00'.
  • BOOL ou BOOLEAN: Armazena valores booleanos (verdadeiro ou falso). Também podem ser representados como 0 para falso e 1 para verdadeiro.
  • FLOAT: Armazena números decimais de precisão simples.
  • DOUBLE: Armazena números decimais de dupla precisão, com mais precisão que FLOAT.

Estes são apenas alguns dos tipos de dados suportados pelo SQL. A escolha do tipo de dados adequado depende dos requisitos específicos da sua base de dados e da aplicação que está a ser desenvolvida.

Índices

Um índice numa base de dados é semelhante a um índice num livro. É um ponteiro para dados numa tabela. Um índice numa base de dados é uma estrutura de dados que melhora a velocidade das operações de recuperação de dados numa tabela.

Por exemplo, vamos considerar uma tabela Alunos com colunas id, nome e idade. Se você quiser encontrar um aluno com id = 123, sem um índice, o software de base de dados deve analisar cada linha na tabela Alunos - um processo chamado "varredura completa da tabela". Se Alunos tiver milhões de linhas, isso poderia levar muito tempo. Mas com um índice na coluna id, a base de dados pode encontrar o aluno muito mais rapidamente.

Eis como criar um índice na coluna id da tabela Alunos:

CREATE INDEX idx_alunos_id
ON Alunos (id);

Após criar este índice, quaisquer consultas que filtrem por id serão muito mais rápidas.

Views

Uma vista é uma tabela virtual baseada no conjunto de resultados de uma instrução SQL. Uma vista contém linhas e colunas, assim como uma tabela real. Os campos em uma vista são campos de uma ou mais tabelas reais na base de dados.

As vistas podem ser úteis para:

  • Segurança: As vistas podem exibir colunas específicas da tabela com base na condição definida.
  • Simplicidade: Uma vista pode extrair dados de várias tabelas diferentes e apresentá-los como uma única tabela, transformando consultas de várias tabelas em visualizações de uma única tabela simples.

Por exemplo, vamos considerar que temos duas tabelas: Encomendas e Clientes, e queremos criar uma vista que nos mostre todas as encomendas feitas por um cliente específico. Poderíamos criar uma vista assim:

CREATE VIEW EncomendasCliente AS
SELECT Encomendas.IDEncomenda, Clientes.NomeCliente, Encomendas.DataEncomenda
FROM Encomendas
INNER JOIN Clientes ON Encomendas.IDCliente = Clientes.IDCliente;

SELECT *
FROM EncomendasCliente

Isto irá devolver todas as encomendas feitas por João Pedro, combinando dados tanto das tabelas de Encomendas quanto de Clientes.
